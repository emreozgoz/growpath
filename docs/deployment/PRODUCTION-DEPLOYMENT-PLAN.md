### Phase 2: Environment Configuration (Day 6 Afternoon)

#### 2.1 Frontend Environment Variables (Vercel)

```bash
# API Configuration
NEXT_PUBLIC_API_URL=https://growpath-api.railway.app

# NextAuth (if using)
NEXTAUTH_URL=https://growpath.vercel.app
NEXTAUTH_SECRET=<generate-with-openssl-rand-base64-32>

NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY=pk_live_...

# Analytics
NEXT_PUBLIC_VERCEL_ANALYTICS_ID=<auto-generated>
```

**Generate Secrets**:
```bash
# NextAuth Secret
openssl rand -base64 32

# JWT Secret
openssl rand -base64 64
```

#### 2.2 Backend Environment Variables (Railway/Render)


### Phase 3: Database Migration (Day 6 Evening)

#### 3.1 Run Prisma Migrations

**Option 1: Local Migration (Recommended)**
```bash
# Set production DATABASE_URL
export DATABASE_URL="postgresql://postgres:[password]@..."

# Run migrations
cd backend
npx prisma migrate deploy

# Verify schema
npx prisma db pull
```

**Option 2: Railway/Render Console**
```bash
# SSH into deployed backend
railway run bash  # or Render console

# Run migrations
npx prisma migrate deploy
```

**Verification**:
```bash
# Check tables created
npx prisma studio
```

---

#### 3.2 Seed Initial Data (Optional)

**Create seed script**: `backend/prisma/seed.ts`
```typescript
import { PrismaClient } from '@prisma/client'

const prisma = new PrismaClient()

async function main() {
  // Create test user
  const user = await prisma.user.create({
    data: {
      email: 'demo@growpath.com',
      name: 'Demo User',
      password: '<hashed>', // Use bcrypt
      tier: 'FREE'
    }
  })

  console.log('Seeded:', user)
}

main()
  .catch(console.error)
  .finally(() => prisma.$disconnect())
```

**Run seed**:
```bash
npx prisma db seed
```

## 🔒 Security Checklist

### Pre-Deployment Security

- [ ] All secrets in environment variables (not in code)
- [ ] `.env` files in `.gitignore`
- [ ] CORS configured (only allow production frontend)
- [ ] Rate limiting enabled
- [ ] SQL injection protection (Prisma ORM)
- [ ] XSS protection (React auto-escaping)
- [ ] HTTPS enforced (automatic on Vercel/Railway)
- [ ] Security headers (helmet.js)

### Post-Deployment Security

- [ ] Change all default passwords
- [ ] Rotate API keys if previously exposed
- [ ] Enable Supabase Row Level Security (RLS)
- [ ] Set up Vercel WAF (free tier)
- [ ] Monitor Sentry for security errors
- [ ] Review npm audit vulnerabilities
